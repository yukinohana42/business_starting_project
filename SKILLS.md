# Skill一覧

このプロジェクトで使うAIエージェント用スキルの一覧です。

## workshop-designer

新しいワークショップ回を設計するときに使います。

出力は、目的、対象、伝える概念、導入、メインワーク、問い、まとめ、宿題、必要なカードやワークシートです。

## slide-writer

子供向け、または親向けのMarkdownスライド原稿を作るときに使います。

1スライド1メッセージ、短い問い、身近な例を重視します。

## card-designer

印刷用カード教材を作るときに使います。

価値カード、問題カード、顧客カード、アイデアカード、PMF信号カード、ピッチ問いカードなどを作ります。

## parent-briefing-writer

親向け説明資料を作るときに使います。

起業を押し付けるものではなく、価値を作る力を育てるプロジェクトとして説明します。

## facilitator-script-writer

親が当日読める進行台本を作るときに使います。

親が話しすぎず、子供が考え、話し、選ぶ時間を作ることを重視します。

## gamification-designer

教材にミッション、カード、進捗、バッジ、Final Missionなどを入れるときに使います。

ゲーミフィケーションは学びの足場として扱い、勝敗、順位、報酬で子供を動かさないことを重視します。

## character-illustration-designer

人物イラストやオリジナルキャラクターを教材に入れるときに使います。

完全オリジナルキャラクターを使い、既存作品や特定漫画家の絵柄を模倣せず、価値を人物の変化として見せます。

## visual-storytelling-designer

抽象的な概念を、図解、Before/After、マンガ的1コマ表現、短い吹き出しで見せるときに使います。

Zoomで読める文字量と、親が見ても安心できる表現を重視します。

## visual-learning-system-designer

子供向け資料を、説明スライドではなく発見ゲーム型の学習体験として設計するときに使います。

必ず `docs/visual_learning_system_v2.md`、`docs/global_visual_acceptance_criteria.md`、`docs/session_visual_design_map_v2.md` を参照します。

主な仕事:

- 各ページの画面タイプを Mission Board / Case File / Card Table / Experiment Lab などに分類する。
- キャラクターが何をしているかをページごとに決める。
- ゲーム要素がどの学習行動を進めるかを決める。
- 丸アイコンだけ、ゲーム用語だけ、カードを並べただけの資料を不合格にする。

## parent-material-designer-v2

親向け資料を、落ち着いたが実用的なFacilitator Guideとして設計するときに使います。

必ず `docs/parent_material_design_principles_v2.md` を参照します。

主な仕事:

- 親を先生、採点者、ゲームマスターとして描かない。
- Question Card、OK/NG Response Sheet、Safety Boundary Card、Praise Guideを入れる。
- 結果ではなく、観察、問い、試したこと、改善を支える資料にする。
