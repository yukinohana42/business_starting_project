# Global Visual Learning System Prompt

Use this prompt before creating any child-facing or parent-facing visual material for the junior entrepreneurship workshop.

## Required Reading

Read these first:

- `SOURCE_OF_TRUTH.md`
- `DESIGN.md`
- `docs/visual_learning_system_v2.md`
- `docs/global_visual_acceptance_criteria.md`
- `docs/session_visual_design_map_v2.md`
- `docs/parent_material_design_principles_v2.md`

## First Classification

Classify the material as one of these:

- Child-facing infographic or slide
- Child-facing card
- Child-facing worksheet
- Parent-facing guide or slide
- Parent-facing card or checklist

## Child-Facing Rule

If the material is child-facing, define before writing or designing:

- Screen concept: Mission Board, Quest Map, Detective Note, Case File, Evidence Board, Card Table, Experiment Lab, Pitch Stage, Final Mission Sheet, or Reflection Log.
- Character action: what the character is doing.
- Game element: which card, map, note, stamp, badge, or sheet is active.
- Learner action: what the child sees, chooses, writes, says, or tests.
- Visible change: Before/After or learning change.
- Safety rule: no ranking, scoring, reward bait, money hype, or existing IP imitation.

Do not proceed if the result is only a heading, body text, and cards.

## Parent-Facing Rule

If the material is parent-facing, define before writing or designing:

- Parent role: facilitator, not teacher, scorer, or game master.
- Parent action: ask, observe, support, protect safety, praise process.
- Materials: Question Card, OK/NG Response Sheet, Safety Boundary Card, Session Flow Map, Observation Log, Praise Guide, or Reflection Support Sheet.
- Safety rule: no anxiety, no entrepreneurship pressure, no money hype, no comparison between children.

## Stop Conditions

Stop and return a redesign plan if:

- Characters are round icons, faces only, name labels only, or standing only.
- Game terms appear only in titles.
- Cards are decorative and do not create learner action.
- Parent materials make parents evaluators or game masters.
- The material is a normal explanation slide.
- Acceptance criteria have not been filled.

## Required Output Before Production

Before HTML/CSS, PNG, PDF, PPTX, or print layout generation, output an acceptance table with:

- Character action
- Game element function
- Learner or parent action
- Visible change
- Safety
- Readability
