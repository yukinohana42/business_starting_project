# Day3 Visual Design Brief v2

## Classification

- Audience: 子供向け
- Session: Day3 Idea Builder / アイデアビルダー
- Status: 制作準備のみ。HTML/CSS/PNGはまだ作らない。

## Screen Concept

Idea Workshop / Card Table / Customer-Problem-Solution Board

## Mission

誰の何を助けるかを形にせよ。

## Character Actions

- ソラが顧客カード、困りごとカード、解決案カードを机に並べる。
- ユウが「誰がどう変わる？」と確認する。
- ソラがラフスケッチを描く。
- 子供キャラが組み合わせを選ぶ。

## Game Elements

- Customer Card
- Problem Card
- Solution Card
- Value Change Card
- Idea Builder Board
- Draft Stamp

## Visible Change

- ひとりで困る -> 助けてもらえる
- わからない -> できそう
- 面倒 -> ラク
- つまらない -> 楽しい

## Learner Actions

- 顧客カードを選ぶ。
- 困りごとカードを並べる。
- 解決案カードを作る。
- 「誰の何を何に変えるか」を1文で言う。

## Prohibitions

- すごいアイデアだけを褒める。
- 儲かりそうかで評価する。
- 事業計画書のように重くする。
- 普通の説明表にする。
- 丸アイコンだけのソラにする。

## Acceptance Criteria

- Card Tableとして見える。
- ソラがカードやスケッチを使って考えを形にしている。
- 子供が選ぶ/組み合わせる行動が見える。
- 顧客、問題、解決案が相手の変化につながっている。

## Human Review Before Production

- アイデアの数より、相手の変化が主役になっているか。
- カードの種類が多すぎないか。
- 画面がワークショップ感を持っているか。
