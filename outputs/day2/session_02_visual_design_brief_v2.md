# Day2 Visual Design Brief v2

## Classification

- Audience: 子供向け
- Session: Day2 Problem Hunter / 困りごとハンター
- Status: 制作準備のみ。HTML/CSS/PNGはまだ作らない。

## Screen Concept

Problem Hunt / Daily Observation Map / Question Card

## Mission

日常に隠れた価値の種を見つけろ。

## Character Actions

- ユウが学校、家、部活、スマホ時間の観察マップを見ている。
- ミホが「いつ困る？」「今はどうしている？」のQuestion Cardを出す。
- 子供キャラが付箋に困りごとを書く。
- 安全なインタビュー相手を選ぶ。

## Game Elements

- Problem Card
- Observation Map
- Question Card
- Interview Safety Card
- Discovery Stamp
- Problem Hunter Log

## Visible Change

- 不満 -> 気づき
- 面倒 -> 解決の種
- 好き -> もっと欲しい
- なんとなく困る -> 具体的に言える

## Learner Actions

- 身近な場面を選ぶ。
- 困りごとを付箋に書く。
- 質問カードから1つ選ぶ。
- 安全な範囲のインタビュー相手を決める。

## Prohibitions

- 困っている人を笑う。
- 個人情報を聞き出す。
- 無理にインタビューさせる。
- 親が「それはビジネスにならない」と潰す。
- 普通の困りごと説明スライドにする。
- 丸アイコンだけのキャラクター。

## Acceptance Criteria

- 観察マップが画面上で機能している。
- Problem Cardに書く/選ぶ行動がある。
- ミホの問いが安全なインタビューへつながる。
- 子供が次に何を観察するかがわかる。
- 勝敗、採点、報酬化がない。

## Human Review Before Production

- インタビュー安全境界が十分か。
- 「不満」「面倒」「好き」を中2が自然に扱えるか。
- 画面が説明ではなくObservation Mapに見えるか。
