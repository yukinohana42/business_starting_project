# gamification-designer

## 用途

ワークショップや教材に、学びを助けるゲーミフィケーションを入れるときに使う。

このSkillは、子供を競わせたり、報酬で釣ったりするためのものではない。ミッション、カード、進捗、振り返りを使って、発見・仮説・実験・改善を進めやすくする。

## 必ず読むもの

- `SOURCE_OF_TRUTH.md`
- `DESIGN.md`
- `docs/gamification_design.md`
- `docs/gamification_mechanics_catalog.md`
- `docs/session_gamification_map.md`
- 対象セッションの `README.md`

## 出力形式

- ミッション名
- その回のゴール
- 子供に残したい一言
- 使うカード
- 使うワーク
- 使えるバッジやスタンプ
- 親の問い
- やってはいけない演出
- Final Mission

## ルール

- ゲーミフィケーションは学びの足場であり、目的ではない。
- 勝敗、順位、ランキングを安易に入れない。
- 報酬より、発見と振り返りを重視する。
- バッジやポイントは評価ではなく、学びの行動の記録にする。
- 子供向け資料では「問い」「選択」「発見」「小さな挑戦」を入れる。
- 親はゲームマスターではなく、問いを出す伴走者として扱う。

## 品質基準

- `SOURCE_OF_TRUTH.md` と矛盾しない。
- 起業を押し付けない。
- 不安を煽らない。
- 点数で子供を序列化しない。
- 中学2年生が照れずに参加できる言葉にする。
- みんほす！らしい温かさを保つ。
