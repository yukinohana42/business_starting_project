# Image Generation Visual Review Checklist

## Purpose

画像生成後のcandidateを、教材として採用してよいか判断するための共通チェックリスト。

読みやすい、雰囲気がよい、かわいい、だけでは合格にしない。発見ゲーム型教材として、子供が何をするかが見え、価値が相手の変化として伝わる必要がある。

## 1. 教材意図との一致

- ページの学習目的が絵で伝わるか。
- 子供が何をする画面かわかるか。
- Mission / Case / Card / Map などの画面タイプが見えるか。
- キャラクターが学習行動をしているか。
- ゲーム要素が学習行動と結びついているか。
- 価値を「相手の変化」として見せているか。

## 2. 意味不明な描写の検出

以下があれば修正対象:

- 小物が意味不明な場所から出ている。
- 持ち物が手とつながっていない。
- 虫眼鏡、ペン、カード、ノートなどが顔や体に不自然に重なる。
- 同じ小物が不要に増殖している。
- 背景装飾が学習意図より目立つ。
- 何をしている場面かわからない。
- キャラクターの視線と行動が合っていない。
- 手、指、腕、姿勢が不自然で学習行動に見えない。
- カードやノートが読めないほど崩れている。
- 日本語文字が崩れて意味不明。
- 画面内のラベルが内容と矛盾している。
- お金や報酬を煽るように見える。
- 勝敗、ランキング、採点、トロフィー、レベルアップ報酬のように見える。

## 3. キャラクター品質

- ユウ、ソラ、ミホが完全オリジナルに見えるか。
- 中2が見ても幼すぎないか。
- 親が見ても安心できる服装・表情か。
- キャラクターが飾りではなく、観察、メモ、指差し、カード操作、問いかけなどをしているか。
- ページごとの役割が明確か。
- 顔アイコンや立っているだけの人物になっていないか。
- 既存キャラクター、既存作品、特定絵柄に似ていないか。

## 4. ゲーミフィケーション品質

- ミッション感があるか。
- ただの説明スライドになっていないか。
- スタンプやバッジが報酬ではなく学びの記録に見えるか。
- Quest MapやMission Cardが実際の学習進行と結びついているか。
- 派手なゲームUIや勝敗演出になっていないか。
- ミッション、探偵、Finalなどの言葉だけでゲーム化した状態になっていないか。

## 5. 文字と後編集

- 画像内の日本語が崩れていないか。
- 細かい文字を画像生成に任せすぎていないか。
- 後載せすべき文字が画像内に入りすぎていないか。
- 最終教材として使う場合、どの文字を後載せするべきか。
- 大見出し、短いラベル、空欄、カード文言が後編集しやすい余白を残しているか。
- 画像生成で崩れた文字を教材本文として使おうとしていないか。

## 6. 採用判定

| Decision | Use When |
| --- | --- |
| PASS_FINAL | そのままfinal候補としてよい |
| PASS_WITH_MINOR_FIX | 小修正または後載せで対応可能 |
| REGENERATE_WITH_REVISED_PROMPT | プロンプトを修正して再生成が必要 |
| REJECT | 方針違い、教材として不適切、または意味不明描写が多く不採用 |

## 7. Review Note Template

```markdown
# Image Review

- page id:
- candidate:
- prompt path:
- reviewer:
- visual review availability:
- decision:

## Good Points

## Issues

## Meaningless / Broken Depictions

## Text Handling

## Safety Check

## Correction Prompt

## Final / Rejected Reason

## Next Action
```

## 8. Stop Conditions

以下の場合はfinal化しない。

- Codexも人間も画像を視覚確認していない。
- キャラクター行動が不明。
- 小物の位置や意味が教材理解を邪魔する。
- Before/Afterが不明。
- 文字崩れを教材本文として使う必要がある。
- 既存IP模倣、お金煽り、勝敗、採点に見える。
