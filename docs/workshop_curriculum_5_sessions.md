# 5回構成カリキュラム

## 全体像

標準は5回構成です。各回30から60分で実施できます。

| 回 | テーマ | ゴール |
| --- | --- | --- |
| 1 | 働く・お金・価値 | 仕事は価値提供だとわかる |
| 2 | 価値の見つけ方 | 不満、面倒、好きから価値の種を見つける |
| 3 | アイデア作り | 誰の何を解決するかを言える |
| 4 | PMF | 作りたいものと欲しいものの違いを知る |
| 5 | ピッチ | 3分でアイデアを説明できる |

## 第1回

問い: 一番ラクに稼いでいそうな仕事は何？

学び: 仕事は物を売ることではなく、誰かの状態を良くすること。

ワーク: 価値変換ゲーム。困ったを助かったに変える、面倒をラクに変える、退屈を楽しいに変える。

## 第2回

問い: 最近イラッとしたこと、面倒だったことは何？

学び: ビジネスの種は、日常の不満、面倒、好き、もっと楽しくしたい気持ちの中にある。

ワーク: 3人にインタビューする質問を作る。

## 第3回

問い: その困りごとは、誰が一番困っている？

学び: 良いアイデアは「誰に、どんな価値を、どう届けるか」がはっきりしている。

ワーク: 10分で20個の解決案を出し、困り度、頻度、試しやすさで選ぶ。

## 第4回

問い: 無料なら使う人は、本当に欲しい人？

学び: PMFとは、作ったものと欲しい人の気持ちがピタッとはまること。

ワーク: PMF信号を青、黄、赤で判断し、小さな実験を設計する。

## 第5回

問い: セミナーで発表するなら、何を一番伝える？

学び: すごさではなく、誰のどんな問題をどう解くかが伝わることが大事。

ワーク: 3分ピッチを作り、親が審査員として質問する。

## v2追加: 視覚教材化の共通方針

各回を資料化するときは、普通の説明スライドにしない。`docs/session_visual_design_map_v2.md` に沿って、各回を発見ゲーム型の教材画面にする。

- Day1: Detective Mission / Case File / Value Change Card
- Day2: Problem Hunt / Daily Observation Map / Question Card
- Day3: Idea Workshop / Card Table / Customer-Problem-Solution Board
- Day4: Mini Experiment Lab / Signal Board / Reaction Log
- Day5: Pitch Stage / Story Board / Listener Memo

各回で、キャラクター行動、ゲーム要素、子供の行動、相手の変化を定義してから制作する。

注: Day5の親は「審査員」ではなく、問いを出す聞き手・伴走者として扱う。
