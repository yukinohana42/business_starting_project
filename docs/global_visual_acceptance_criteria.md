# Global Visual Acceptance Criteria

## Purpose

この基準は、Day1からDay5、親向け説明資料、カード、ワークシート、4:5版、A4印刷物の共通合格基準です。

「読みやすい」だけでは合格ではありません。発見ゲーム型の学習体験として機能していることが必要です。

## 1. Child-Facing Infographics and Slides

各ページごとに以下を確認します。

| 項目 | 合格 | 不合格 |
|---|---|---|
| キャラクター | 具体的な行動をしている | 丸アイコン、顔だけ、立っているだけ |
| ゲーム要素 | 学習行動と結びついている | タイトルにゲーム用語があるだけ |
| 変化 | 相手のBefore/Afterが見える | 概念説明だけ |
| 子供の行動 | 見る、選ぶ、書く、話すが明確 | 読むだけ |
| 画面 | Mission / Case / Card / Mapとして見える | 見出し、本文、カードだけ |
| 安全性 | 勝敗、採点、報酬化がない | 点数、ランキング、ご褒美感がある |
| 可読性 | Zoom/印刷で読める | 小さすぎる、詰め込みすぎ |

## 2. Parent-Facing Materials

| 項目 | 合格 | 不合格 |
|---|---|---|
| 親の役割 | ファシリテーターとして描く | 先生、採点者、ゲームマスターとして描く |
| 声かけ | 問い、観察、伴走が中心 | 正解提示、評価、説教が中心 |
| 褒め方 | 行動プロセスを褒める | 結果、すごさ、儲かりそうを褒める |
| 安全性 | 起業を押し付けない | 将来不安や競争で煽る |
| デザイン | 落ち着いたが温かい | 無機質すぎる、子供向けゲームUIすぎる |
| 実用性 | そのまま進行に使える | 理念だけで具体的行動がない |

## 3. Cards

カード教材は、次の条件を満たします。

- 1カード1行動。
- 子供が選ぶ、並べる、書き込む、比較する、話すために使える。
- カードが単なるイラストや飾りではない。
- カード名が行動を示している。
- 正解カードではなく、問いや観察の入口である。

推奨カード:

- 観察カード
- 困りごと発見カード
- 価値変換カード
- 顧客カード
- 実験カード
- 反応メモカード
- ピッチ問いカード

## 4. Worksheets

ワークシートは、次の条件を満たします。

- 子供が自分の言葉で書ける。
- 正解欄ではなく観察欄になっている。
- 親が採点する形式になっていない。
- Before/After、誰が、どう変わる、次に何を試す、が書ける。
- 最後にFinal Missionを書く欄がある。
- 書きやすく、余白が十分ある。

## 5. Production Stop Conditions

以下のいずれかに該当する場合は、HTML/CSSやPNG生成へ進みません。

- キャラクターの行動がページごとに定義されていない。
- ゲーム要素がページごとに定義されていない。
- 子供が何をする画面なのかが定義されていない。
- 親向け資料で親が採点者になっている。
- ゲーミフィケーションが報酬や勝敗に見える。
- 単なる説明スライドになっている。
- 固定方針との整合チェック表がない。

## 6. Review Required Questions

制作物レビューでは、必ず次を問います。

- これは説明資料か、発見ゲーム型教材画面か。
- キャラクターは内容を動かしているか。
- ゲーム要素は子供の行動を進めているか。
- 画面を見た子供が次に何をするかわかるか。
- 親が評価者にならない設計か。
- 可読性のために体験設計を捨てていないか。
