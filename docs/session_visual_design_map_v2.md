# Session Visual Design Map v2

## Purpose

Day1からDay5、親向け説明資料まで、すべての教材を同じ思想で作るための視覚設計マップです。

## Day1: Value Detective / 価値探偵

- テーマ: 働く・お金・価値
- ミッション: お金と仕事の正体を見破れ
- 画面コンセプト: Detective Mission / Case File / Value Change Card
- 主役キャラクター: 価値探偵ユウ

### Character Actions

- 観察する
- 探偵ノートにメモする
- Before/Afterを見比べる
- 価値変換カードを作る

### Game Elements

- Mission Card
- Detective Note
- Evidence Card
- Value Change Card
- Discovery Stamp
- Final Mission Sheet

### Change

- 困った -> 助かった
- 面倒 -> ラク
- 退屈 -> 楽しい
- わからない -> できそう

### Final Mission

仕事とは、＿＿を＿＿に変えること。

## Day2: Problem Hunter / 困りごとハンター

- テーマ: 価値の見つけ方
- ミッション: 日常に隠れた価値の種を見つけろ
- 画面コンセプト: Problem Hunt / Daily Observation Map / Question Card
- 主役キャラクター: 価値探偵ユウ + ナビゲーター・ミホ

### Character Actions

- 日常の困りごとを観察する
- 付箋に困りごとを書く
- 「いつ困る？」「今はどうしている？」を聞く
- 安全な範囲でインタビュー相手を選ぶ

### Game Elements

- Problem Card
- Observation Map
- Question Card
- Interview Safety Card
- Discovery Stamp
- Problem Hunter Log

### Change

- 不満 -> 気づき
- 面倒 -> 解決の種
- 好き -> もっと欲しい
- なんとなく困る -> 具体的に言える

### Final Mission

価値の種は、＿＿の中にある。

### Safety Notes

- 困っている人を笑わない。
- 個人情報を聞き出さない。
- 無理にインタビューしない。
- 親が「それはビジネスにならない」と潰さない。

## Day3: Idea Builder / アイデアビルダー

- テーマ: アイデア作り
- ミッション: 誰の何を助けるかを形にせよ
- 画面コンセプト: Idea Workshop / Card Table / Customer-Problem-Solution Board
- 主役キャラクター: アイデアビルダー・ソラ

### Character Actions

- 顧客カードを選ぶ
- 困りごとカードを並べる
- 解決案カードを作る
- ラフスケッチする
- 「誰がどう変わるか」を1文にする

### Game Elements

- Customer Card
- Problem Card
- Solution Card
- Value Change Card
- Idea Builder Board
- Draft Stamp

### Change

- ひとりで困る -> 助けてもらえる
- わからない -> できそう
- 面倒 -> ラク
- つまらない -> 楽しい

### Final Mission

私たちは、＿＿の＿＿を＿＿に変える。

### Safety Notes

- 「すごいアイデア」だけを褒めない。
- 儲かりそうかで評価しない。
- 具体的な誰かに向ける。
- 会社員や起業しない選択を悪く見せない。

## Day4: Experiment Designer / 実験デザイナー

- テーマ: PMFと小さな実験
- ミッション: 本当に欲しい人がいるか小さく試せ
- 画面コンセプト: Mini Experiment Lab / Signal Board / Reaction Log
- 主役キャラクター: ソラ + ミホ

### Character Actions

- 小さく試す方法を選ぶ
- 反応を見る
- PMF信号カードを並べる
- 赤信号を改善メモに変える
- 次の小さな実験を考える

### Game Elements

- Experiment Card
- PMF Signal Card
- Reaction Log
- Red Signal Improvement Card
- Mini Test Map
- Learning Stamp

### Change

- 欲しいかわからない -> 反応が見える
- 作りっぱなし -> 試して改善する
- 赤信号 -> 次の改善ヒント
- 思い込み -> 相手の反応

### Final Mission

最初に試す一番小さいことは、＿＿。

### Safety Notes

- 赤信号を失敗扱いしない。
- 無理に販売させない。
- 完璧な完成品を求めない。
- 親が評価者にならない。

## Day5: Pitch Challenger / ピッチチャレンジャー

- テーマ: 3分ピッチ
- ミッション: 3分で価値の変化を伝えろ
- 画面コンセプト: Pitch Stage / Story Board / Listener Memo
- 主役キャラクター: ユウ + ソラ + ミホ

### Character Actions

- 価値の変化を1文にまとめる
- 問題、相手、変化、試し方を並べる
- 発表する
- 聞き手メモを見る
- 改善点を1つ選ぶ

### Game Elements

- Pitch Card
- Value Story Board
- Listener Memo Card
- Final Quest Map
- Reflection Stamp
- Next Challenge Card

### Change

- 何をするかわからない -> 誰の何をどう良くするかが伝わる
- アイデアだけ -> 価値の変化として伝わる
- 発表して終わり -> 聞いた反応で改善する

### Final Mission

私たちのアイデアは、＿＿を＿＿に変える。

### Safety Notes

- 発表の勝ち負けにしない。
- 親の採点会にしない。
- うまく話せたかより、相手の変化が伝わったかを見る。

## Parent Materials

- 目的: 親が先生、採点者、ゲームマスターにならず、問いを出す伴走者として関われるようにする。
- 画面コンセプト: Parent Facilitator Guide / Question Card / OK-NG Response Sheet / Safety Boundary

### Parent Actions

- 先に正解を言わない。
- 子供の観察を聞く。
- 誰がどう変わるかを問い返す。
- 安全な範囲を守る。
- 結果ではなく過程を褒める。
- 反応が悪いときに改善へつなげる。

### Elements

- Facilitator Map
- Question Card
- OK/NG Voice Card
- Safety Boundary Card
- Praise Guide
- Reflection Support Sheet

### Prohibitions

- 起業しないと困ると煽る。
- 稼げそうかで評価する。
- すごい/すごくないで評価する。
- 親の好みの答えに誘導する。
- 兄弟や他人と比べる。
- 発表を採点する。

## Manual Handoff Application

画像生成を使う場合は、全セッションでmanual handoff modeを標準にします。Codexがprompt packetとレビュー運用を作り、ユーザーがGPT画像生成へpromptを貼ります。

生成物はすべてcandidateから始め、レビューと人間承認なしにfinal化しません。このルールは4:5スマホ共有版、A4印刷物、カード、ワークシート、親向け説明資料にも適用します。
