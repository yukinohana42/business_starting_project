# Global Visual System Spec

## Purpose

Day1からDay5、親向け資料、カード、ワークシート、スライド、インフォグラフィックで共通して使うビジュアルシステムを定義する。

このシステムは、画像生成を前提にする。HTML/CSSは、画像を補助し、文字とレイアウトを整えるために使う。

## Core Tone

- 温かい。
- 親しみやすい。
- 中学2年生向けだが幼稚すぎない。
- 整理されている。
- Zoomで読める。
- 不安を煽らない。
- 起業を押し付けない。
- お金を下品に扱わない。

## Color System

| Role | Color Direction | Use |
| --- | --- | --- |
| Base paper | warm cream | 背景、ノート、カード下地 |
| Main text | soft navy | 見出し、本文、重要語 |
| Discovery | warm yellow | 発見、スタンプ、気づき |
| Value / help | soft green | 助かった、安心、価値変化 |
| Structure | pale blue | マップ、整理、ケース線 |
| Question | coral | 問い、注意、ミホの問いカード |
| Parent | calm gray-blue | 親向け資料の落ち着き |

Avoid:

- 強い原色の多用。
- ネオン色。
- 高級感や金儲け感の強い金色。
- 全体が単色に見える配色。

## Character System

### Yuu / Value Detective

Role:

- 価値探偵。
- 誰がどう変わるかを観察する。

Visual cues:

- 中学2年生に近い年齢感。
- 普通のカジュアル服。
- 探偵ノート、ペン、ミッションカード、虫眼鏡。

Actions:

- 観察する。
- メモする。
- 証拠カードを貼る。
- Before/Afterを指差す。

### Sora / Idea Builder

Role:

- アイデアを形にする。
- カード、付箋、ラフスケッチを使う。

Visual cues:

- 中学2年生に近い年齢感。
- カジュアル服。
- 付箋、カード、ペン、ラフスケッチ、試作メモ。

Actions:

- カードを並べる。
- 解決案を描く。
- 小さな実験を選ぶ。
- 反応ログを書く。

### Miho / Navigator

Role:

- 正解を言わず、問いを出す。
- 親と子供の間に立つ案内役。

Visual cues:

- 年上すぎない。
- 落ち着いて親しみやすい。
- Question Card、Safety Boundary Card、Reflection Card。

Actions:

- 問いカードを差し出す。
- 安全境界を示す。
- 振り返りを促す。

## Character Rules

- 完全オリジナルキャラクターにする。
- 既存IP、既存キャラクター、特定漫画家の絵柄を模倣しない。
- 顔アイコンだけは禁止。
- 立っているだけは禁止。
- キャラクターは学習行動の担い手として描く。
- 表情だけでなく、手元、姿勢、視線、小物で行動を見せる。

## Screen Types

### Mission Board

Use:

- Day1導入、Day2導入、Day5全体確認。

Must show:

- 今日のミッション。
- キャラクターがカードやノートを見ている。
- 最後に書く文やゴール。

### Case File

Use:

- Day1仕事・お金・身近な例。

Must show:

- Evidence Card。
- 誰がどう変わるか。
- ユウが観察またはカードを貼る。

### Evidence Board

Use:

- Day1お金、Day2観察、Day4反応。

Must show:

- 証拠を貼る、比べる、線でつなぐ。

### Card Table

Use:

- Day1価値変換、Day3アイデア作り。

Must show:

- 机。
- 手元。
- カードを選ぶ、並べる、書き込む。

### Detective Note

Use:

- Day1、Day2。

Must show:

- 観察メモ。
- 「誰が」「どう変わる」などの短い問い。

### Experiment Lab

Use:

- Day4。

Must show:

- 実験カード。
- 信号カード。
- 反応ログ。
- 赤信号が改善メモへ変わる流れ。

### Pitch Stage

Use:

- Day5。

Must show:

- 発表する子供。
- 聞き手メモ。
- 価値の変化カード。
- 勝敗や採点ではなく、反応と改善。

### Parent Facilitator Guide

Use:

- 親向け資料。

Must show:

- 親が子供を評価する場面ではなく、問いを出し、横で伴走する場面。
- OK/NG Voice Card。
- Safety Boundary Card。

| Session | Main Visual Mode | Main Character | Key Action |
| --- | --- | --- | --- |
| Day1 | Mission Board / Case File / Card Table | Yuu | 誰がどう変わるかを観察する |
| Day2 | Problem Hunt / Observation Map | Yuu + Miho | 日常の困りごとを見つける |
| Day3 | Idea Workshop / Card Table | Sora | 顧客・問題・解決案を組み合わせる |
| Day4 | Experiment Lab / Signal Board | Sora + Miho | 小さく試して反応を見る |

## Text Handling

Image generation should not carry the full Japanese copy.

Use generated image for:

- scene
- character
- props
- emotion
- board/card shapes

Use layout after generation for:

- exact Japanese text
- blanks
- page numbers
- small labels
- card copy
- worksheet fields

## Prop Handling

小物は、学習行動と結びつく場合だけ使う。

Allowed when meaningful:

- magnifying glass held by Yuu or placed clearly on a table
- pen connected to hand or notebook
- detective notebook held, open, or placed on the work surface
- mission card held, pinned, or placed on Mission Board
- stamp slot placed as a record area, not a reward
- Quest Map placed as progress context

Disallow:

- floating props
- duplicated unnecessary props
- props emerging from face, hair, body, or background
- props covering eyes, face, hands, or key text areas
- decorative props that do not support seeing, choosing, writing, talking, comparing, or trying
- meaningless labels or broken generated Japanese used as final text

Every candidate image must be reviewed for prop meaning before final adoption.

## Global Negative Prompt

- existing anime, manga, game, mascot, brand, logo, or character resemblance
- imitation of a specific artist, studio, or franchise
- money piles, luxury cars, expensive watches, rich lifestyle
- winner, loser, ranking, score, trophy, prize, reward bait
- scary future warning, pressure to start a company
- childish preschool style
- face-only icons
- characters standing without action
- cluttered poster with too much text
- generic business seminar slide
- cold corporate stock art

## Acceptance Check

Every generated visual must answer:

1. What page type is this?
2. Who is doing what?
3. What game element supports the action?
4. What does the learner do next?
5. What change is visible?
6. Is any text better added after image generation?
7. Does it avoid entrepreneurship pressure, money hype, competition, and IP imitation?
8. Are props physically meaningful and connected to the learning action?
9. Has the candidate passed visual review before being marked final?
