# Parent Material Design Principles v2

## Purpose

親向け説明資料、親チェックリスト、親用問いカード、親向け進行台本の共通ルールです。

親向け資料は、子供向けのゲームUIをそのまま移植しません。落ち着いた見た目にしながら、親がその場で使えるFacilitator Guide、Question Card、OK/NG Response Sheet、Safety Boundaryとして設計します。

## Parent Role

- 親はファシリテーター。
- 親は採点者ではない。
- 親はゲームマスターではない。
- 親は先生として正解を教えるのではなく、問いを出す伴走者。

親が先に正解を言うと、子供は自分の観察より「親が喜ぶ答え」を探し始めます。

## Core Questions

親の問いは、次を中心にします。

- 誰がどう変わる？
- 今はどうしている？
- 何が困っている？
- それが変わると何がうれしい？
- 小さく試すなら？
- 反応が悪かったらどこを変える？

## Praise Rules

褒める対象は、結果ではなくプロセスです。

### Praise

- 観察したこと。
- 問いを作ったこと。
- 相手を具体的にしたこと。
- 小さく試そうとしたこと。
- 反応を見て改善を考えたこと。
- 自分の言葉でまとめたこと。

### Avoid

- 儲かりそう。
- 一番すごい。
- 起業家っぽい。
- ほかの子より良い。
- 正解。
- 親の好みに近い。

## Required Material Forms

親向け資料には、理念だけでなくその場で使える部品を入れます。

- Question Card
- OK/NG Voice Card
- Session Flow Map
- Safety Boundary Card
- Observation Log
- Praise Guide
- Reflection Support Sheet

## Design Tone

- 子供向けほどゲームUIを強くしない。
- 無機質な説明資料にしない。
- 温かく、落ち着きがあり、実施時に迷わない。
- 1ページに情報を詰め込みすぎない。
- 親が読み上げやすい短い文を使う。

## Safety Boundaries

- 起業を押し付けない。
- 不安を煽らない。
- お金で煽らない。
- 子供を他人や兄弟と比べない。
- 失敗、赤信号、反応が薄いことを責めない。
- インタビューや実験は家庭内で安全な範囲にする。

## Acceptance Check

親向け資料を作る前に、次を確認します。

- 親の次の一言が具体的に書かれているか。
- OK例とNG例があるか。
- 親が採点者に見えないか。
- 親がゲームマスターに見えないか。
- 起業やお金で煽っていないか。
- 子供の観察、問い、試行、改善を支える内容か。
