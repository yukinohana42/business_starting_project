# ビジュアルストーリーテリング方針

## 基本方針

言葉だけで説明せず、人物の変化で価値を見せます。価値は「商品」や「サービス名」そのものではなく、受け取った人の状態が少し良くなることです。

## 抽象概念を具体例に落とす

抽象語を使ったら、必ず生活に近い例に戻します。

- 価値提供 -> 困ったを助かったに変える。
- 顧客 -> それを本当に欲しい具体的な人。
- PMF -> 作ったものと欲しい人の気持ちがピタッとはまること。
- 実験 -> 完璧に作る前に、小さく試してみること。

## 価値を人物の変化で見せる

- 困った顔 -> 助かった顔。
- だるそう -> ラクそう。
- 退屈 -> ワクワク。
- わからない -> できそう。
- ひとりで困る -> 友達と盛り上がる。

この変化は、表情だけでなく、姿勢、行動、手元のメモ、周囲との会話でも見せます。

## Before/Afterのルール

- Beforeは困っている状態を短く見せる。
- Afterは少し良くなった状態を見せる。
- 相手を笑いものにしない。
- 大げさな成功表現にしない。
- 1つの変化に絞る。

## 1ページあたりの人物数

- 基本は1から2人。
- 多くても3人まで。
- Page 5のような例一覧では、人物を増やしすぎず、カードや小さな表情アイコンで補う。
- 1ページに人物が多い場合は、主役を1人に決める。

## 1ページあたりのイラスト数

- 基本はメインイラスト1つ。
- 補助イラストは1から2つまで。
- イラストを増やすより、図解、カード、矢印で意味を整理する。
- 4:5版に展開するときは、イラストを縦に積める構成にする。

## 図解とイラストのバランス

- イラストは感情や場面を見せる。
- 図解は関係や流れを見せる。
- カードは子供が考える入口を見せる。
- 矢印は変化を見せる。

例:

- 「面倒 -> ラク」は表情の変化と矢印で見せる。
- 「ありがとう -> お金」は人の表情と小さな流れ図で見せる。
- 「仕事とは何か」は人物イラストと価値変換カードを組み合わせる。

## Zoomで見たときの読みやすさ

- 小さい表情に頼りすぎない。
- 表情、言葉、矢印を組み合わせる。
- 吹き出しは短くする。
- 人物の細部より、姿勢と変化が見える構図にする。
- 画面共有で読めない小さな文字を入れない。

## 安全な見せ方

- 失敗や困りごとを笑わない。
- 不安を煽らない。
- 起業やお金を過度に派手に見せない。
- 会社員や既存の仕事を下げない。
- 親は評価者ではなく、問いを出す伴走者として見せる。

## Day1で重視する変化

- 仕事: 困っている人が少し助かる。
- お金: ありがとうが見える形になる。
- 価値: 困った、面倒、退屈が少し良くなる。
- 身近な例: コンビニ、ラーメン屋、塾、動画、ゲームを「相手の変化」で見る。
- まとめ: 自分の言葉で、仕事を変化として言う。

## v2追加: 場面として見えること

「カードが並んでいる」だけでは、マンガ的1コマや発見ゲーム型教材としては不十分です。キャラクターがカードを作る、指差す、相談する、メモするなど、場面として見える必要があります。

不合格になりやすい例:

- 見出し、本文、カードだけで画面が完結している。
- Before/Afterが顔アイコンだけで、状況や行動がない。
- 探偵ノートがあるが、何を観察しているかが見えない。
- Final Mission欄があるだけで、書いている場面がない。

制作前に、各ページを「誰が、どこで、何をしている場面か」で1文にします。
