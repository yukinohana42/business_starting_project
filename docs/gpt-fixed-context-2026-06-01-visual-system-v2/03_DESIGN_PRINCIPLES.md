# 教材設計方針

## 教育設計

このワークショップは、起業の知識を暗記する授業ではありません。子供たちが自分の生活の中から「誰かにとって価値があること」を見つけ、相手に聞き、小さく試すための練習です。

中心に置く学びは次の3つです。

- 観察する: 困りごと、面倒、好き、欲しい気持ちを見つける
- 聞く: 思い込みではなく、相手の話から考える
- 試す: 完璧に考えず、小さく動いて反応を見る

## プロジェクト全体のビジュアルトーン

このプロジェクトの見た目は、無機質な教育資料やAIが自動生成したような冷たい資料にしません。

目指す印象は、みんほす！で普段使うような、温かく、親しみやすく、やわらかい、でも見やすく整理された教材です。中学生が「ちょっと話してみてもいいかも」と感じ、親御さんが「安心して見せられる」と感じるトーンにします。

キーワード:

- 温かい
- やわらかい
- 親しみやすい
- 教育的で信頼感がある
- 子供っぽすぎない
- 整理されているが冷たくない
- Zoomで見ても読みやすい

## みんほす！らしい温かいデザイン方針

みんほす！らしさは、派手さではなく「安心して読める」「自然に話したくなる」「人に寄り添っている」印象として扱います。

具体策:

- 角の強い四角ではなく、少し丸みのある枠を使う
- 余白を広めに取り、詰め込み感を避ける
- 見出しはやさしいネイビーや落ち着いた色で統一する
- 強調色は1画面に1から2色までにする
- イラストは意味がある場所だけに使う
- アイコンや図解は「困った→助かった」の変化を助けるために使う

## 中学2年生向けの設計

- 身近な題材を使う
- 正解探しではなく、発見ゲームにする
- 1つの問いに対して複数の答えがあってよい形にする
- 競争やランキングを安易に使わず、発見と振り返りを中心にする
- 書く量は多すぎないようにし、話す、選ぶ、カードを動かす活動を入れる
- 子供向けでも幼稚にはしない

## ゲーミフィケーションを入れたビジュアル方針

ゲーミフィケーションは、学びをミッション化して発見しやすくするために使います。ゲーム化そのものを目的にせず、問い、選択、発見、小さな挑戦、振り返りを見える形にします。

使ってよい要素:

- ミッションカード
- ビジネス探偵ノート
- クエストマップ
- バッジ
- ステッカー
- 進捗ゲージ
- Final Mission

ただし、安っぽいゲームUI、派手なネオン表現、勝敗を煽る演出、ランキング風の見せ方は避けます。みんほす！らしい温かさを保ちながら、少しだけ冒険感、探偵感、発見感を出します。

## オリジナルキャラクターの使い方

人物イラストは、価値や変化を伝えるために使います。完全オリジナルの同世代キャラクターを使い、既存作品、既存キャラクター、特定漫画家の絵柄は模倣しません。

推奨する役割:

- 価値探偵ユウ: 誰がどう変わるかを観察する。
- アイデアビルダー・ソラ: カードやラフスケッチで考えを形にする。
- ナビゲーター・ミホ: 正解ではなく、短い問いを出す。

キャラクターは中学2年生が感情移入しやすいが、幼稚にしすぎません。親が見ても安心できる服装、表情、言葉にします。

## マンガ的1コマ表現の使い方

マンガ的な1コマ表現は、理解補助として使います。1ページに1から2コマまでとし、吹き出しは短くします。擬音は多用しません。探偵ノート、ミッションカード、変化カードなど、教材独自の視覚言語を使います。

## Before/Afterイラストの使い方

価値は、人物の変化として見せます。

- 困った顔 -> 助かった顔
- だるそう -> ラクそう
- 退屈 -> ワクワク
- わからない -> できそう
- ひとりで困る -> 友達と盛り上がる

Beforeは相手を笑いものにせず、Afterは大げさな成功ではなく、少し良くなった状態にします。

## Zoom画面共有に最適化した見せ方

Zoomで見せる資料は、印刷物よりもさらに文字量を絞ります。

ルール:

- 16:9を基本にする
- 1画面1メッセージを基本にする
- 本文は1画面あたり3から5行まで
- 小さい注釈や長い説明文を入れない
- 画面共有時に読める大きな文字サイズを前提にする
- 重要語は色と太さで軽く強調するが、太字を使いすぎない
- 親が話すための補足はスライドに詰めず、台本側に置く

## 1画面あたりの推奨情報量

子供向けスライド:

- 見出し: 1つ
- メインメッセージ: 1つ
- 例: 1つ、多くても2つ
- 問い: 1つ

インフォグラフィック:

- 大きなタイトル: 1つ
- 1ページの主メッセージ: 1つ
- 1ページの本文: 3から5行まで
- 1ページの図解: 1つ、多くても2つ
- 全体枚数: 説明の流れに合わせて4から6枚
- 図解: 変化を表す矢印やカードを中心にする

親向け資料:

- 1ページに入れる説明は、目的、進め方、親の役割の3ブロック程度まで
- 不安を煽る言葉を入れない
- 家庭でそのまま使える問いを入れる

## 文字・フォントの方針

- 丸みのある、やさしい印象の日本語フォントを前提にする
- 可読性を最優先する
- 見出しと本文のメリハリをつける
- 太字を使いすぎない
- 子供向けでも幼稚な丸文字に寄せすぎない
- Zoom画面共有でも読める文字サイズを前提にする
- 長い文章より、短い問いと短い例を使う

推奨イメージ:

- 見出し: やさしい太さのゴシック
- 本文: 読みやすい標準ゴシック
- 強調: 色、余白、配置で行い、太字だけに頼らない

## 色の方針

白背景でもよいですが、真っ白で冷たくしません。クリーム系や淡い色面を使い、温かさを出します。

推奨色:

- やわらかいネイビー: 見出し、本文の基調
- 落ち着いたコーラル: 注意や問いのアクセント
- あたたかいイエロー: 発見、気づき
- やさしいグリーン: 価値、助かった、安心
- 淡いブルー: 説明、整理、Zoom共有背景
- クリーム系背景: 全体の温かさ

避けること:

- 原色を強く並べる
- 全体を単一色だけで支配する
- 子供っぽすぎるカラフルさにする
- 大事でない部分まで色で強調する

## イラスト・図解の方針

- シンプルで意味のあるイラストを使う
- かわいすぎず、親しみがある
- フラットであたたかい印象にする
- 「仕事＝誰かの状態を良くする」「困った→助かった」「面倒→ラク」など、意味が伝わる図解を優先する
- 飾りだけのイラストは増やしすぎない
- Zoomで見ても一目で意味がわかることを優先する

## スライド設計

- 1スライド1メッセージ
- 1枚に文章を詰め込みすぎない
- できるだけ問いで始める
- 例は「YouTuber」「ゲーム実況」「ラーメン屋」「コンビニ」「部活」「テスト」「文化祭」などを使う
- 専門用語を出す場合は、次の行で必ず中学生向けに言い換える
- スライドは進行を前に進める役割、インフォグラフィックは全体理解を支える役割にする

## 複数枚インフォグラフィック設計

インフォグラフィックは、Zoomで説明しながら概念を理解するための資料です。1枚に全部を詰め込まず、説明の流れに合わせて4から6枚程度に分けます。

スライドは進行を前に進める役割、インフォグラフィックは概念理解と全体整理を支える役割です。

ルール:

- 1ページ1メッセージを守る
- 1ページに全部を詰め込まない
- 導入、仕事、お金、価値、身近な例、まとめのように説明順で分ける
- 各ページの役割を明確にする
- 「困った→助かった」「面倒→ラク」「退屈→楽しい」のような変化を大きく見せる
- 本文を増やしすぎず、図解と短いコピーで伝える
- 16:9のZoom共有を主用途にする
- 将来4:5に展開できるよう、ページ単位で縦積みにしやすい構成にする
- 親が横で見ても、論理の流れが追えるようにする

## カード教材設計

カードは、考えるきっかけをランダムに引ける道具です。親が説明しすぎず、子供がカードを選び、並べ、組み合わせるために使います。

カードの種類は次の通りです。

- 価値カード
- 問題カード
- 顧客カード
- アイデアカード
- PMF信号カード
- ピッチ問いカード

## ワークシート設計

ワークシートは、完成度の高い事業計画書ではなく、考えを見える形にするためのメモです。

- 空欄は短く書ける大きさにする
- 最初から完璧に埋めなくてよい
- 「わからない」と書ける欄を残す
- 最後に「次に試すこと」を必ず入れる
- Zoom実施時は、手元で紙に書くか、チャットに短く書ける形にする

## 親向け設計

親向け資料では、教育的な意味と家庭での関わり方を説明します。親が答えを出すのではなく、問いを出し、子供の観察を助けることを重視します。

親向け資料のデザインは、子供向けより落ち着かせます。ただし冷たくせず、温かさと安心感を残します。

## AIが作った感じを避ける具体策

- 抽象的な大見出しだけで終わらせない
- 「価値を創出しましょう」のような硬い言葉を避ける
- 実際の会話で使う言葉に寄せる
- 身近な例を必ず入れる
- 図解は意味のあるものだけにする
- 不自然に整いすぎた箇条書きだけにしない
- 親が読み上げても自然な文にする
- 子供が答えやすい問いにする

## 親と子供の両方に伝わる注意点

- 子供には短い問いと例で伝える
- 親には、その問いの教育的意味がわかるように補足する
- 子供を焦らせる言葉を避ける
- 親が評価者になりすぎないようにする
- 起業をゴールにせず、価値を見つける力をゴールにする

## Visual Learning System v2 追加ルール

今後の子供向け資料は、単なる説明スライドではなく、発見ゲーム型の教材画面として設計します。詳しくは `docs/visual_learning_system_v2.md` と `docs/global_visual_acceptance_criteria.md` を参照します。

必須:

- 画面は Mission Board / Quest Map / Detective Note / Case File / Evidence Board / Card Table / Experiment Lab / Pitch Stage / Final Mission Sheet / Reflection Log のどれかとして設計する。
- キャラクターはメモする、観察する、指差す、カードを並べる、付箋を書く、ラフスケッチする、質問する、試す、反応を見る、振り返る、発表するなどの行動をしている。
- ゲーム要素はタイトル語や飾りではなく、子供の「見る、選ぶ、書く、話す、試す」を進めるUIとして機能している。
- 価値や学びは、相手のBefore/Afterまたは学びの変化として見える。

禁止:

- 見出し、本文、カードを並べただけの普通の説明スライド。
- 丸アイコン、顔だけ、名前ラベルだけのキャラクター。
- 「ミッション」「探偵」「Final Mission」などの語を置くだけ。
- 可読性だけを満たして、発見ゲーム型の体験設計を満たしていない状態。

HTML/CSSやPNG生成へ進む前に、ページごとのキャラクター行動、ゲーム要素、子供の行動、相手の変化、合格基準表を作ること。
