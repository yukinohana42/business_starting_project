# Agents Rules

## 基本ルール

CodexやAIがこのプロジェクトで作業するときは、先に `02_SOURCE_OF_TRUTH.md` を読みます。

作業前に、必ず project root、branch、HEAD、worktree status、remote URL を確認します。GitHubに関わる作業では、remote branchのHEADも確認します。

`02_SOURCE_OF_TRUTH.md` を最優先します。ほかのファイルと矛盾した場合は、SOURCE_OF_TRUTHを優先し、必要なら矛盾を報告します。

## 守ること

- 中学2年生が読んで意味がわかる言葉にする
- 親向け資料では落ち着いた説明にする
- 子供向け資料では短い問い、身近な例、選べるワークを中心にする
- 「起業しなければいけない」という押し付けをしない
- 「会社員は危険」「サラリーマンは終わり」など、不安を煽る表現を避ける
- 抽象論だけで終わらせず、生活に近い例に戻す
- docs-only作業と教材生成作業を分ける
- 大規模変更は作業ブランチで行う
- mainへの直接作業は、ユーザーが明示した場合に限る
- ブラウザプロファイル、Cookie、Login Data、Cache、Crashpad dump、`.env`、秘密情報をcommitしない
- 画像やキャラクターは、既存漫画・アニメの固有キャラ、ロゴ、固有の絵柄を再現しない

## 書き方

- 1文を短くする
- 難しい言葉は言い換える
- 「顧客」は「困っている具体的な人」と説明する
- 「PMF」は「作ったものと欲しい人の気持ちがピタッとはまること」と説明する
- 「失敗」は「方向を変えるサイン」と説明する

## セッションの基本形

1. 目的
2. 今日わかること
3. 導入の問い
4. メインワーク
5. 発表または共有
6. まとめ
7. 宿題または次回までの観察

## 変更時の確認

- `SOURCE_OF_TRUTH` と矛盾していないか
- 子供向けと親向けのトーンが混ざっていないか
- スライド、ワークシート、カード、台本の内容が同じ方向を向いているか
- 次に直すことを `TODO.md` または `17_CURRENT_STATUS_AND_NEXT_TASKS.md` に反映するか
- docs-only commitに、教材生成物、画像生成物、PDF、PPTX、ブラウザプロファイルが混ざっていないか
