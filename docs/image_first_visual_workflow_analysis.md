# Image-First Visual Workflow Analysis

## Purpose

Day1 v2 / v3 prototypeは、読めるZoom資料としては成立したが、意図していた「発見ゲーム型の教材画面」には届かなかった。

今後は、Day1だけでなくDay2からDay5、親向け資料まで、HTML/CSSで絵を直描きする流れを標準にしない。教材の中心となる場面、人物、手元、小物、学習行動は、画像生成または専用イラスト素材を前提に設計する。

## Root Cause

### 1. HTML/CSS中心だった

HTML/CSSは、文字、余白、カード、枠、書き出しには強い。しかし、発見ゲーム型教材に必要な次の表現には弱かった。

- キャラクターの視線、手元、姿勢、道具の持ち方。
- ユウ、ソラ、ミホが学習行動を担っている場面。
- 机の上でカードを並べる、ノートに書く、証拠カードを貼るなどの具体動作。
- Before/Afterの人物変化を自然な1コマとして見せること。
- Mission Board、Case File、Card Tableなどが本当に「場面」として見えること。

その結果、画面は「説明見出し + 本文 + カード + 簡易人物」になりやすかった。

## 2. Acceptance Criteria Were Too Weak

既存の受入基準は、次をよく見ていた。

- Zoomで読めるか。
- 文字が重ならないか。
- 1ページ1メッセージか。
- 安全な表現か。

これらは必要だが、十分ではなかった。

不足していた確認は次の通り。

- これは説明スライドではなく、教材画面として見えるか。
- キャラクターが学習行動を実際に動かしているか。
- ゲーム要素が子供の「見る、選ぶ、話す、書く、比べる、試す」に結びついているか。
- 画面の中心が文字説明ではなく、場面・行動・変化になっているか。
- 画像生成前に、キャラクター、場面、手元、小物、学習行動のpromptが定義されているか。

## 3. Characters Were Not Functional Enough

ユウ、ソラ、ミホの名前と役割は正しい。しかし、v2 / v3試作ではキャラクターがページを動かすほど機能しなかった。

不十分だった状態:

- 顔アイコン、簡易人物、立っている人物に近い。
- 手元の作業が弱い。
- 何を見て、何を書いて、何を選んでいるかが曖昧。
- ゲーム要素とキャラクター行動が分離している。

合格する状態:

- ユウは観察し、メモし、証拠カードを貼る。
- ソラはカードを並べ、ラフスケッチし、小さく試す手順を作る。
- ミホは正解を言わず、問いカードや安全境界カードを差し出す。
- キャラクターの視線、手、道具が、子供の次の行動を示している。

## 4. Game Elements Became Labels

Mission Card、Detective Note、Evidence Card、Discovery Stampなどは登場したが、機能より見た目の名前になりやすかった。

今後は、各ゲーム要素について必ず次を定義する。

| Game Element | Required Function |
| --- | --- |
| Mission Card | 今日する行動と最後に書く文へつなぐ |
| Detective Note | 「誰が、どう変わる？」を記録する |
| Evidence Card | 身近な例のBefore/Afterを観察する |
| Value Change Card | もとの状態と変わった後を組み合わせる |
| Discovery Stamp | 発見した行動の記録であり報酬ではない |
| Reflection Stamp | 振り返りの印であり採点ではない |
| Final Mission Sheet | 自分の言葉でまとめる |
| Question Card | 親やミホが正解ではなく問いを出す |

## 5. Image-First Correction

今後の標準は、次の順番にする。

1. 学習目的を整理する。
2. 画面タイプを決める。
3. キャラクター行動を1ページ1文で決める。
4. ゲーム要素が支える学習行動を決める。
5. 相手のBefore/Afterまたは学びの変化を決める。
6. 画像生成promptを作る。
7. key visualを生成する。
8. 文字とカードを後編集またはHTML/CSSで載せる。
9. Zoom/印刷で確認する。

HTML/CSSは、主役の絵を作る手段ではなく、次の補助として扱う。

- 文字載せ。
- カードや空欄の再配置。
- 16:9 / 4:5 / A4への整形。
- contact sheet作成。
- final export。

## Production Stop Rule

画像生成promptに次が入っていない場合、生成へ進まない。

- ページタイプ。
- キャラクターの行動。
- 手元と小物。
- ゲーム要素の機能。
- 子供または親の行動。
- Before/Afterまたは学びの変化。
- 禁止表現。
- 文字を画像内に詰め込まない方針。

## What Not To Repeat

- HTML/CSSだけで全ビジュアルを完成させようとする。
- 読めるだけで合格にする。
- キャラクターを顔アイコンや記号として扱う。
- Mission、Detective、Finalなどの語を置くだけでゲーム化したと判断する。
- 画像生成前に、画面の場面設計を省略する。
