# Agents Rules

## 基本ルール

- 先に `02_SOURCE_OF_TRUTH.md` を読む。
- 作業前に project root、branch、HEAD、worktree status、remote URL を確認する。
- `SOURCE_OF_TRUTH` を最優先する。
- 大規模変更は作業ブランチで行う。
- docs-only作業と教材生成作業を分ける。

## 守ること

- 中学2年生が読んで意味がわかる言葉にする。
- 親向け資料では落ち着いた説明にする。
- 子供向け資料では短い問い、身近な例、選べるワークを中心にする。
- 起業を押し付けない。
- 不安を煽らない。
- お金を下品に扱わない。
- 価値を相手の変化として扱う。
- 可能な限り各セッションにミッションを設定する。
- ミッション化によって学習目的を薄めない。
- 競争やランキングを安易に入れない。
- 子供向け資料では「問い」「選択」「発見」「小さな挑戦」を入れる。
- イラストは飾りではなく、価値や変化を伝えるために使う。

## 禁止・注意

- ブラウザプロファイル、Cookie、Login Data、Cache、Crashpad dump、`.env`、秘密情報をcommitしない。
- PNG再生成が不要なdocs-only作業で画像を作らない。
- 既存作品、既存キャラクター、特定作品の絵柄を模倣しない。
- バッジやポイントを能力評価として使わない。
- PNG、HTML、CSSを固定情報源フォルダへコピーしない。必要な場合はMarkdownにパスと説明だけ書く。
- `git add .` は使わず、存在する対象パスだけを明示的にstageする。

## 変更時の確認

- `SOURCE_OF_TRUTH` と矛盾していないか。
- 子供向けと親向けのトーンが混ざっていないか。
- 次に直すことを `TODO.md` に反映したか。
- docs-only commitに画像生成物、PDF、PPTX、ブラウザプロファイルが混ざっていないか。

## 成果物作成後の更新ルール

- 新しい方針を追加したら、`SOURCE_OF_TRUTH.md`、`DESIGN.md`、`AGENTS.md`、`SKILLS.md` のどこへ反映すべきか確認する。
- 次タスクが変わったら、`TODO.md` と固定情報源内の `14_CURRENT_STATUS_AND_NEXT_TASKS.md` を更新する。
- 固定情報源を更新したら、`docs/GPT_FIXED_CONTEXT_CURRENT.md` が最新版フォルダを指しているか確認する。
