# Visual Storytelling Guidelines

## 基本方針

言葉だけで説明せず、人物の変化で価値を見せます。価値は商品やサービス名そのものではなく、受け取った人の状態が少し良くなることです。

## 抽象語の言い換え

- 価値提供 -> 困ったを助かったに変える。
- 顧客 -> それを本当に欲しい具体的な人。
- PMF -> 作ったものと欲しい人の気持ちがピタッとはまること。
- 実験 -> 完璧に作る前に、小さく試してみること。

## ページ設計

- 1ページ1メッセージ。
- 人物は基本1から2人、多くても3人。
- メインイラスト1つ、補助イラスト1から2つまで。
- 図解は関係や流れを見せる。
- カードは子供が考える入口を見せる。
- 矢印は変化を見せる。

## マンガ的1コマ表現

- 理解補助として使う。
- 1ページに1から2コマまで。
- 既存漫画作品の表現やキャラクターを模倣しない。
- 探偵ノート、ミッションカード、変化カードなど、教材独自の視覚言語にする。
- 子供っぽくなりすぎず、親が見ても安心できる表現にする。

## 吹き出し

- 1つの吹き出しは1文までを基本にする。
- 中学生がそのまま言いそうな言葉にする。
- 親向け説明を吹き出しに詰め込まない。
- 例: 「誰が助かった？」「まず小さく試してみよう」「自分の言葉でOK」。

## Zoomでの読みやすさ

- 小さい表情に頼りすぎない。
- 表情、言葉、矢印を組み合わせる。
- 吹き出しは短くする。
- 画面共有で読めない小さな文字を入れない。

## Day1 v2への反映

- Page 1: 今日のミッションを1コマで見せる。
- Page 2: 困った顔から助かった顔へのBefore/Afterを大きく見せる。
- Page 3: ありがとうが届く流れを人物と図解で見せる。
- Page 4: 価値変換カードを並べる手元を見せる。
- Page 5: ビジネス探偵ノートで身近な仕事を観察する。
- Page 6: Final Missionを書き込む場面を見せる。
